## 二、各模块设计

### ① 词汇模块（升级现有卡片）
现有卡片太单调，升级为「信息更丰富的词卡」：

- **词性标签**（你要的）：n. 名词 / v. 动词 / adj. 形容词 / adv. 副词 …
  - 数据来源：ECDICT 自带 `pos` 字段 + 释义前缀，可直接解析。✅ 容易
  - 不同词性用不同颜色小标签，一眼区分。
- **例句**（你要的）：每个词配 1–2 句真实例句 + 中文翻译 + 朗读。
  - 数据来源见第三节。
- **词形变化**：复数 / 过去式 / 比较级等（ECDICT `exchange` 字段有）。✅ 容易
- **「我背过的词」列表**（你要的）：可浏览、搜索、按状态筛选（新词 / 学习中 / 已掌握 / ⭐收藏），点进去看详情、重听。✅ 容易
- 记忆曲线、自评、发音：已实现，保留。

### ② 语法模块（新）
0 基础最缺"规则"。做成**循序渐进的小课程**：

- 课程顺序（示例）：be 动词 → 名词单复数 → 人称代词 → 一般现在时 → 冠词 a/an/the → 一般过去时 → 现在进行时 → 情态动词 …
- 每课结构：**口语化讲解（中文）→ 例句 → 小测（选择 / 填空 / 连词成句）→ 过关**。
- 内容来源：我来编写一套结构化课程（可借助 AI 生成初稿，我审校），存成本地数据。✅ 中等（主要是写内容）

### ③ 跟读 / 发音矫正模块（新）⚠️ 技术最难
- **跟读**：播放标准发音 → 你录音 → 回放对比。✅ 容易（浏览器录音）
- **发音评分 / 矫正**：判断你读得准不准、哪个音错了。这是**整个产品技术门槛最高的部分**，有三档方案：

  | 方案 | 能力 | 成本 | 适配 |
  |------|------|------|------|
  | A. 浏览器语音识别 | 只能判断"读对/读错"(整词) | 免费 | 入门够用，精度有限 |
  | B. 微软 Azure 发音评估 | **逐个音素打分** + 准确度/流利度/完整度 | 有免费额度，超出按量付费 | 专业级，**推荐** |
  | C. 科大讯飞评测 | 类似 B，国内访问快 | 付费 | 国内可选 |

  > 说人话：**免费版**只能告诉你"这个词读得不对，再来一次"；**专业版（Azure）**能告诉你"th 这个音发错了，得咬舌"。后者需要接入云服务（涉及一点配置和少量费用）。

### ④ 日常运用模块（新）
让单词语法"用起来"：

- **情景对话**：点餐、问路、购物、自我介绍… 每个场景一组对话 + 跟读 + 关键句。
- **AI 对话练习**（强烈推荐）：和 AI 用英语聊天，它会配合你的水平、即时纠错、给出更地道的说法。
  - 技术：调用 Claude API。需要联网 + 一点后端（见第四节）+ 少量 API 费用。
- **每日一句 / 实用短语**：每天推一句地道表达。

### ⑤ 打卡 / 激励系统（贯穿）
- **连续打卡天数（streak）** + 日历热力图（哪天学了一目了然）。
- **每日目标**：今日新词 / 复习 / 跟读完成度。
- **提醒**（你要的）：
  - 网页版：靠浏览器通知（需用户授权，手机锁屏时不一定可靠）。
  - 真·手机 App：才有稳定的定时推送。→ 这是后期"打包成真 App"会解决的。

## 三、各模块数据来源汇总

| 内容 | 来源 | 状态 |
|------|------|------|
| 单词 + 音标 + 中文 | ECDICT | ✅ 已就绪（3344 词）|
| 词性 / 词形变化 | ECDICT `pos` / `exchange` 字段 | ✅ 直接解析 |
| 例句 + 翻译 | **Tatoeba**（开源句库，英中句对，部分带真人音）或 **AI 生成** | 待定（见决策）|
| 语法课程 | 自编（可 AI 起草 + 审校）| 待做 |
| 情景对话 | 自编（可 AI 起草）| 待做 |
| 标准发音 | 浏览器 TTS（现有）→ 后期可换真人音 | ✅ 有 |
| 发音评分 | Azure 发音评估 / 浏览器识别 | 待定（见决策）|
| AI 对话 | Claude API | 待定（见决策）|

**例句两种来源对比：**
- **Tatoeba（静态句库）**：免费、离线、句子真实自然，但不一定每个词都有、难度参差。
- **AI 生成**：每个词都能定制、可控难度（专为 0 基础写简单句），但要调 API、有费用、需联网。
- 也可**二者结合**：优先用 Tatoeba，缺的用 AI 补。

## 四、技术架构演进（关键）

现在的"单个 HTML 文件 + 本地存储"撑不住新需求了。但**不需要一步到位**，分三阶段：

### 阶段 1：纯前端多页面版（不花钱、可离线、马上能做）
- 把 App 拆成多页面（底部导航：单词/语法/跟读/运用/我的）。
- 词汇升级（词性 + 例句[Tatoeba] + 词形 + 背词列表）、语法课、情景课、打卡系统、跟读录音对比。
- 数据全部打包进本地，进度存浏览器本地存储。
- **不需要服务器、不需要 API、不花钱。** 内容已经很丰富。

### 阶段 2：加基础语音（免费）
- 用浏览器自带的语音识别做跟读"读对/读错"判断。

### 阶段 3：上云服务（解锁 AI + 专业发音，需少量配置和费用）
- 加一个**轻量后端 / 云函数**（保护 API 密钥，前端直连会泄露密钥不安全）。
- 接 **Claude API**（AI 对话、AI 例句）+ **Azure 发音评估**（逐音素打分）。
- 这步我会手把手带你配，费用是"按用量"的小钱（有免费额度兜底）。

### 阶段 4（可选）：打包成真·手机 App
- 用 PWA 或跨平台框架把网页版打包，解锁稳定推送、上架应用商店。

> **建议**：先把**阶段 1** 做扎实（零成本、内容丰富、马上能用），你用一段时间验证方向，再决定要不要上阶段 3 的 AI / 专业发音。

## 五、建议的开发路线图

1. **第一步**：重构成多页面框架 + 底部导航（搭好骨架）。
2. **第二步**：词汇模块升级（词性、例句、词形、背词列表）。
3. **第三步**：打卡 / 进度 / 提醒系统。
4. **第四步**：语法模块（先出 5–8 课打底）。
5. **第五步**：跟读模块（录音 + 回放对比 + 免费识别判断）。
6. **第六步**：情景对话模块。
7. **第七步（进阶）**：上云，接 AI 对话 + Azure 发音评分。
